Django Template Engine Overview

Django's default template engine is designed to separate presentation from business logic, allowing for dynamic HTML generation. It uses a syntax that is easy to read and understand.

{% verbatim %}
Feature Description Example
Default Template Engine Django's template engine allows for dynamic HTML generation using template files with placeholders for variables and control structures, promoting clean code separation. <h1>{{ title }}</h1> (where `title` is passed from the view)
Blocks Blocks are sections defined in templates that allow for inheritance and overriding. A base template can define blocks, and child templates can fill or override these blocks, enhancing code reusability. {% block content %}...{% endblock %}
Loops (for ... in) The `for ... in` tag is used to iterate over lists or dictionaries. It enables the dynamic rendering of repeated elements, like displaying items in a list or rows in a table. {% for item in item_list %}<p>{{ item }}</p>{% endfor %}
If Control Structures The `if` tag allows for conditional rendering. It enables the display of certain content based on specific conditions, increasing the template's adaptability. {% if user.is_authenticated %}<p>Welcome, {{ user.username }}!</p>{% endif %}
Display of Context Variables Context variables passed from views can be accessed using double curly braces `{{ variable_name }}`, enabling dynamic content rendering based on server-side data. <h2>{{ page_title }}</h2> (where `page_title` is passed from the view)
